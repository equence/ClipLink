# ClipLink：跨平台局域网剪贴板同步工具

## 目标

用 C++ 开发一个跨平台的局域网剪贴板同步工具。用户在一台设备复制纯文本后，其他已连接的设备自动收到相同文本，并可在图形界面中查看最近同步历史。

项目用于展示 TCP 客户机—服务器模型、持续 Socket 连接、事件驱动 GUI 与跨平台编译，不复用实验报告中已有的事件管理软件内容。

## 技术方案

- 语言：C++17。
- GUI 与跨平台能力：Qt 6 Widgets。
- 网络：Qt Network（`QTcpServer` 与 `QTcpSocket`）。
- 构建：CMake。
- 目标平台：Windows、macOS、Linux；演示环境优先 Windows + MSVC。
- 数据格式：长度前缀加 UTF-8 JSON，避免 TCP 粘包/半包问题。

不加入图片、文件同步、账号系统、云端服务或端到端加密，以保持实验范围清晰。

## 架构

```text
ClipLink Client A ─┐
ClipLink Client B ─┼── TCP ── ClipLink Server ── 广播给房间内其余客户端
ClipLink Client C ─┘
```

### 服务端

服务端没有图形界面也可独立运行，负责监听端口、记录已连接设备与转发消息。收到合法的剪贴板消息后，为其分配序号并转发给除发送者以外的客户端。服务端不永久保存剪贴板内容。

### 客户端

客户端分为三部分：

- `ClipboardWatcher`：用 `QClipboard` 监听本机纯文本复制事件，并写入收到的远端文本。
- `NetworkClient`：维护 TCP 连接、处理协议帧、重连和心跳。
- `MainWindow`：显示状态、历史记录和设置，调用前两者但不直接处理 Socket 细节。

每台客户端启动时生成 `deviceId`；每条消息携带 UUID。客户端维护近期 UUID 缓存，收到已处理消息时跳过，以避免“远端写入剪贴板—本地监听再次上传”的回环。

## GUI 设计

采用简洁的浅色桌面工具风格，而不是密集的表单界面：

- 顶部：应用名、圆点连接状态、当前服务器地址，以及一个明显的“暂停同步”开关。
- 主区域：左侧为最近 20 条同步记录；右侧显示选中记录的完整文本、来源设备和时间。
- 底部：手动发送文本输入区与“发送”按钮，便于没有复制操作时演示通信。
- 设置对话框：服务器地址、端口、设备显示名、启动时自动连接。

视觉令牌：使用深靛蓝作为主色、青绿色表示在线、琥珀色表示重连、红色表示错误；窗口圆角 12 px、卡片 8 px、较宽松的 12–16 px 间距。历史记录以首行预览、来源和相对时间呈现，避免大面积边框。图标使用 Qt 自带样式图标或开源 Material Symbols 图标。

深色模式不作为第一版必需功能；若时间允许，可通过 Qt 调色板补充。

## 通信与数据流

1. 客户端连接后发送 `hello`（设备 ID 与显示名）。
2. 用户复制文本，`ClipboardWatcher` 生成 `clipboard` 消息（UUID、来源、时间、文本）。
3. 服务端校验长度和消息类型，按连接广播给其他客户端。
4. 接收客户端先查 UUID 缓存，再写入系统剪贴板并追加历史记录。
5. 断线时客户端界面转为“已断开”，自动尝试有限次数重连；恢复连接后显示成功提示。

单条文本限制为 16 KiB；超长文本提示用户且不发送。

## 错误处理

- 无法连接、服务端关闭或网络中断：状态栏提示；允许手动重连。
- 非法 JSON、未知类型、消息过长：服务端丢弃该帧，不影响其他客户端。
- 剪贴板不可访问：客户端提示权限或平台限制，不退出。
- macOS/Linux 的权限或 Wayland 差异：在 README 中注明，Windows 演示作为验收基准。

## 验收与测试

- Windows 上启动一个服务端、两个客户端；A 复制文本，B 在 1 秒内收到同样文本。
- B 手动发送文本，A 收到并显示来源、时间和内容。
- 反复同步同一条消息不会形成循环或无限历史记录。
- 服务端停止后，客户端显示断线；重新启动后可恢复连接。
- 使用 CMake 在 Windows、macOS、Linux 中至少完成配置与编译验证；功能演示至少在 Windows 完成。
